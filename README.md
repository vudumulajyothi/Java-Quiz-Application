# Java-Quiz-Application
The project supports two ways of creating quiz questions:

Static Input: Questions are hardcoded inside the constructor. This approach is useful for testing and demonstration purposes.
Dynamic Input: Questions are entered by the user at runtime using the Scanner class. This makes the application flexible and allows different quizzes without modifying the source code.



Why keep the static constructor?


✔ Easy for beginners to understand.


✔ Useful for testing the application quickly.


✔ No need to enter questions every time the program runs.


✔ Demonstrates the difference between hardcoded (static) and user-driven (dynamic) data initialization.
